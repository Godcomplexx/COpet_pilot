# ADR-003: Desk-first навигация и независимое состояние сети

## Статус

Принято — 2026-07-16.

## Контекст

CoPet должен ощущаться как живой настольный персонаж, а не устройство, которое
после каждого включения ждёт выбора в главном меню. Сетевое соединение может
меняться независимо от режима UI.

## Решение

- после `BOOT` устройство всегда переходит в `DESK_MODE`;
- прокрутка энкодера в Desk Mode открывает `MENU`;
- короткое касание подтверждает выбор;
- долгое касание из любого режима возвращает в Desk Mode;
- меню закрывается в Desk Mode после 10 секунд без ввода;
- network state хранится отдельно от application mode.

## Обоснование

- лицо появляется сразу после загрузки;
- один понятный жест открывает функции;
- offline/reconnect не размножает состояния UI;
- переходы можно тестировать без железа как чистую логику.

## Компромиссы

- энкодер больше нельзя использовать в Desk Mode без открытия меню;
- режимы, которым нужна прокрутка параметров, должны явно владеть энкодером;
- текущую menu-first реализацию потребуется переносить поэтапно.

## Пересмотреть решение, если

- пользовательские тесты покажут, что случайное открытие меню происходит часто;
- появится отдельная физическая кнопка Menu/Home.
