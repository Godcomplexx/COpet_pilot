# Desk Mode v1 — learning log

## Что мы проверяем

При включении CoPet сразу открывает самостоятельный домашний экран, который
остаётся полезным без телефона и интернета. Экран должен выглядеть живым,
показывать данные комнаты и не мешать работе остальных режимов.

## Задействованные интерфейсы

- ST7789 240×240 по SPI — лицо и карточки состояния;
- SHT31 по I2C — температура и относительная влажность;
- TTP223 на GPIO13 — короткая реакция и долгое возвращение домой;
- энкодер на GPIO32/GPIO33 — открытие и навигация меню.

Распиновка не менялась.

## Реализация

- `main/modes/desk_mode.c` — независимая модель: пороги комфорта, моргание,
  взгляд, лёгкое покачивание и реакция на касание;
- `main/ui/desk_ui.c` — отрисовка лица и карточек прямо в RGB332 framebuffer;
- `main/ui/pip_face_port.c` — C-порт выбранных Pip moods/reactions/vibes;
- `main/app_main.c` — только связывает модель с датчиком, экраном и вводом.

Процедурная анимация не хранит дополнительные полноэкранные кадры. Это важно
для ESP32-WROOM-32 без PSRAM: положение глаз и высота век вычисляются для
текущего момента, после чего рисуются простые фигуры.

Пороги первой версии:

| Состояние | Условие |
|---|---|
| `COLD` | температура ниже 18 °C |
| `HOT` | температура выше 28 °C |
| `DRY` | влажность ниже 30% |
| `HUMID` | влажность выше 70% |
| `COMFY` | остальные корректные значения |
| `SENSOR` | SHT31 не отвечает |

## Управление

- загрузка прошивки → `DESK`;
- короткое касание → радостная реакция;
- вращение колеса → меню;
- короткое касание в меню → открыть выбранный режим;
- долгое касание из меню или другого режима → `DESK`;
- 10 секунд бездействия в меню → `DESK`.

## Как проверить на устройстве

1. Прошить плату и открыть monitor.
2. Убедиться, что в журнале есть `Mode: DESK` и на экране крупное лицо.
3. Дождаться моргания и движения взгляда.
4. Коснуться TTP223: лицо на короткое время улыбается ярче.
5. Повернуть колесо: открывается меню. Ничего не нажимать 10 секунд — снова
   появляется лицо.
6. Согреть SHT31 рукой или аккуратно подышать рядом с ним: значения и цвет
   состояния должны измениться.

## Возможные ошибки

- `SENSOR / NO DATA`: проверить питание SHT31, SDA=GPIO21, SCL=GPIO22 и GND;
- меню открывается само: проверить контакты энкодера и общий контакт COM;
- касание срабатывает постоянно: проверить TTP223 и общий GND;
- анимация дёргается: сначала проверить SPI-проводку, затем уменьшить частоту
  обновления `DESK_FRAME_INTERVAL_US`.

## Результат

ESP-IDF 5.5.2 успешно собрал проект для target `esp32`. Размер приложения
после добавления сенсорных эмоций — `0xAE970`, свободно 32% app-раздела. Прошивка
успешно записана на ESP32 через COM3, хэши проверены. В реальном monitor
подтверждены `Mode: DESK`, чтение SHT31 по адресу `0x44` и событие
`Desk reaction: HAPPY` после короткого касания.

Для C-порта Pip дополнительно проверены реальные переходы: `BORED` после 30
секунд, `SCARED + OVERHEATED` при 28.3 °C, возврат к `BORED` после охлаждения и
`CHILL + SMOKING` после 90 секунд бездействия. MPU6050 корректно отмечен как
отсутствующий, загрузка остальных функций продолжилась.

## Что изучено

- разделение чистой логики режима и аппаратной отрисовки;
- анимация по времени без блокирующих задержек;
- экономия flash/RAM процедурной графикой;
- явные переходы `DESK → MENU → MODE → DESK`.
